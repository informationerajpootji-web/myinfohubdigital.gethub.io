Software क्या है? — और उसके प्रकार
Software निर्देशों (instructions) का वह समूह है जो कंप्यूटर को बताता है कि क्या करना है और कैसे करना है। यह अदृश्य (invisible) होता है — हम इसे देख या छू नहीं सकते, लेकिन इसके बिना Hardware एक निर्जीव मशीन है।
Software की परिभाषा: Software प्रोग्राम्स, डेटा और निर्देशों का वह संग्रह है जो कंप्यूटर hardware को किसी विशेष कार्य को करने योग्य बनाता है। यह Machine Readable Instructions का समूह है जिसे CPU क्रमबद्ध रूप से execute करता है।

Hardware vs Software: Hardware = कंप्यूटर का शरीर (जिसे छू सकते हैं) | Software = कंप्यूटर की आत्मा (जिसे केवल अनुभव कर सकते हैं)
💡 सरल उदाहरण: एक DVD Player में — DVD और Player खुद Hardware है, लेकिन उस Player को चलाने वाला program (जो बटन दबाने पर play करता है, volume बढ़ाता है) — वह Software है।

System Software — सिस्टम सॉफ्टवेयर
Hardware को चलाने वाला, पृष्ठभूमि में काम करने वाला सॉफ्टवेयर
श्रेणी 1 ▼
विस्तृत परिभाषा: System Software वह सॉफ्टवेयर है जो कंप्यूटर के hardware resources को manage और control करता है। यह एक मध्यस्थ (intermediary) का काम करता है — एक तरफ hardware है और दूसरी तरफ Application Software। System Software इन दोनों के बीच संपर्क बनाता है।

यह कंप्यूटर चालू होते ही (boot time) automatically load हो जाता है और पूरे session में पृष्ठभूमि में चलता रहता है। उपयोगकर्ता इससे सीधे interact नहीं करता — लेकिन इसके बिना कुछ भी काम नहीं करेगा।

System Software को Low-Level Software भी कहते हैं क्योंकि यह hardware के बहुत करीब काम करता है। यह Machine Language या Assembly Language में लिखा जाता है ताकि hardware को सीधे नियंत्रित कर सके।

System Software के मुख्य प्रकार

🔵 Operating System (OS)

  • →सबसे महत्वपूर्ण System Software
  • →CPU, RAM, Disk, I/O सब manage करता है
  • →User को GUI या Command Line interface देता है
  • →उदा: Windows 11, Ubuntu Linux, macOS

🔵 Device Drivers (डिवाइस ड्राइवर)

  • →Hardware devices को OS से जोड़ता है
  • →हर device (printer, keyboard, webcam) का अलग driver
  • →बिना driver के hardware काम नहीं करता
  • →उदा: Realtek Audio Driver, Intel HD Graphics Driver

🟣 Utility Programs (उपयोगिता प्रोग्राम)

  • →System की देखभाल और maintenance के लिए
  • →Disk Cleanup — अनावश्यक files हटाता है
  • →Antivirus — virus से सुरक्षा
  • →उदा: CCleaner, Avast, WinRAR, Disk Defragmenter

🟡 BIOS / UEFI (Firmware)

  • →कंप्यूटर ON होने पर सबसे पहले चलता है
  • →POST (Power On Self Test) करता है — hardware जाँचता है
  • →Motherboard की ROM chip में stored होता है
  • →OS को Hard Disk से RAM में load करता है

🔴 Language Translators

  • →High Level Language को Machine Code में बदलते हैं
  • →Compiler, Interpreter, Assembler — तीन प्रकार
  • →ये System Software के हिस्से हैं
  • →उदा: GCC Compiler, Python Interpreter

🟢 Loader & Linker

  • →Program को disk से RAM में load करता है
  • →अलग-अलग program modules को जोड़ता है
  • →Program run होने से पहले काम करता है
  • →OS का अभिन्न हिस्सा

Application Software — एप्लीकेशन सॉफ्टवेयर
उपयोगकर्ता के विशेष कार्यों के लिए — सीधे interact करने वाला
श्रेणी 2 ▼
विस्तृत परिभाषा: Application Software वह सॉफ्टवेयर है जो किसी विशेष उद्देश्य या अंतिम-उपयोगकर्ता (end-user) की जरूरत को पूरा करने के लिए बनाया जाता है। यह System Software के ऊपर (on top of) चलता है। उपयोगकर्ता इससे सीधे interact करता है।

Application Software को App या Program भी कहते हैं। आज हम जो भी काम computer पर करते हैं — document बनाना, internet चलाना, game खेलना, गाना सुनना — ये सब Application Software के माध्यम से होता है।

Application Software High Level Language (Python, Java, C++, JavaScript) में लिखा जाता है। यह OS पर निर्भर है — अगर OS बंद हो तो Application भी बंद हो जाती है। Application Software को End-User Software या Productivity Software भी कहते हैं।

Embedded Software — एम्बेडेड सॉफ्टवेयर
यंत्रों के अंदर छुपा, एक विशेष कार्य के लिए बना सॉफ्टवेयर
श्रेणी 3 ▼
विस्तृत परिभाषा: Embedded Software वह सॉफ्टवेयर है जो किसी विशेष इलेक्ट्रॉनिक उपकरण (device) के अंदर स्थायी रूप से लगाया जाता है। यह उस device की ROM (Read Only Memory) या Flash Memory में संग्रहीत होता है। इसे आम तौर पर Firmware भी कहते हैं।

यह सॉफ्टवेयर केवल एक ही विशेष कार्य के लिए design किया जाता है। जैसे washing machine का software केवल washing cycles control करता है — वह internet चला नहीं सकता, game नहीं खेल सकता।
Embedded Software की विशेषताएँ (Characteristics)
⚡ Real-Time Operation
Real-time में काम करता है। यानी input मिलते ही तुरंत response देना होता है — कोई देरी (delay) नहीं। जैसे Anti-lock Braking System (ABS) को milliseconds में decide करना होता है।
💾 सीमित Memory
बहुत कम memory (KB से MB तक) में काम करता है। इसलिए code बहुत optimized और efficient लिखा जाता है। Assembly Language या C में लिखा जाता है।
🔒 Dedicated Purpose
एक ही काम, हमेशा वही काम। Microwave का software केवल heating control करेगा — कोई अन्य function नहीं। यही इसकी सबसे बड़ी विशेषता है।
⚙️ Hardware-Specific
किसी एक specific hardware के लिए बना होता है। HP Printer का firmware EPSON Printer पर काम नहीं करेगा।
🔄 Firmware Update
कभी-कभी update हो सकता है — जैसे Smartphone का Software Update या Router का Firmware Update। लेकिन पुराने devices में नहीं।
🏃 स्वायत्त (Autonomous)
बिना किसी human intervention के स्वयं काम करता है। Pacemaker को हर धड़कन के लिए user input की जरूरत नहीं।

Proprietary Software — प्रोप्राइटरी सॉफ्टवेयर
बंद स्रोत, लाइसेंस-आधारित व्यावसायिक सॉफ्टवेयर
श्रेणी 4 ▼
विस्तृत परिभाषा: Proprietary Software वह सॉफ्टवेयर है जिसका Source Code (मूल कोड) पूरी तरह गुप्त रखा जाता है और उस पर केवल उसे बनाने वाली कंपनी या व्यक्ति का अधिकार (ownership) होता है। इसे उपयोग करने के लिए एक End User License Agreement (EULA) के तहत license खरीदना पड़ता है।

इसे Closed Source Software या Commercial Software भी कहते हैं। बिना अनुमति के इसे copy करना, share करना, या modify करना Copyright Law के तहत अपराध है।
License के प्रकार
💳 Perpetual License (स्थायी)
एक बार खरीदो, हमेशा उपयोग करो। पुराना MS Office (2019, 2021) ऐसा था। Updates अलग से लेने पड़ते थे।
🔄 Subscription License (सदस्यता)
Monthly या Yearly भुगतान। Microsoft 365, Adobe Creative Cloud। हमेशा latest version मिलता है। भुगतान रुकने पर software बंद।
👥 Per-User License
एक license = एक user। 10 users के लिए 10 licenses। Enterprises में आम। प्रति user ₹ charge होता है।
🖥️ Site License
एक institution / organization को unlimited users के लिए। Schools और companies bulk में खरीदती हैं। सस्ता पड़ता है।
⏱️ Trial / Freemium
Limited time (30 days) या limited features free। पूरी features के लिए pay करो। उदा: WinRAR, Adobe Acrobat, Zoom।
🎁 OEM License
Hardware के साथ बंडल। Windows जो laptop के साथ आता है वह OEM license है। अलग computer पर transfer नहीं होता।

Open Source Software — ओपन सोर्स सॉफ्टवेयर
मुक्त स्रोत, समुदाय-निर्मित, पारदर्शी सॉफ्टवेयर
श्रेणी 5 ▼
विस्तृत परिभाषा: Open Source Software वह सॉफ्टवेयर है जिसका Source Code सार्वजनिक रूप से उपलब्ध होता है। Open Source Initiative (OSI) के अनुसार, कोई भी इसे: (1) देख सकता है (2) उपयोग कर सकता है (3) बदल सकता है (4) वितरित कर सकता है — बिना किसी शुल्क के।

इसे FOSS (Free and Open Source Software) भी कहते हैं। "Free" का मतलब मुफ्त (gratis) नहीं, बल्कि स्वतंत्रता (freedom) है — जैसे "free speech" (स्वतंत्र भाषण) में, न कि "free beer" (मुफ्त बियर) में।
Open Source Licenses के प्रकार
🔓 GPL (GNU General Public License)
सबसे कठोर। यदि आप GPL software में बदलाव करके distribute करते हैं, तो आपका modified version भी GPL होना चाहिए। Linux Kernel GPL है।
🔑 MIT License
सबसे लचीला। लगभग कोई भी प्रतिबंध नहीं। Commercial use में भी उपयोग कर सकते हैं। React.js, jQuery MIT licensed हैं।
🛡️ Apache License 2.0
MIT जैसा flexible। Patents का भी protection देता है। Android, Apache Web Server इसी license पर हैं।
📋 BSD License
बहुत permissive। Attribution जरूरी है। FreeBSD, macOS का Darwin kernel BSD पर आधारित।

Computer भाषाएँ — पीढ़ी दर पीढ़ी विकास
Computer Language वह माध्यम है जिससे हम computer को निर्देश देते हैं। ये निम्न स्तर (Hardware के पास) से उच्च स्तर (मनुष्य के पास) तक क्रमबद्ध हैं। जितना उच्च स्तर, उतना आसान लिखना लेकिन उतनी ही अधिक translation की जरूरत।
Computer Language क्यों चाहिए? CPU केवल Binary (0 और 1) समझता है। लेकिन मनुष्य के लिए 01001000 01100101 01101100... लिखना असंभव है। इसलिए ऐसी भाषाएँ बनाई गईं जो मनुष्य लिख सकें, और फिर उन्हें Machine Language में translate किया जाए। यही Programming Languages का उद्देश्य है।
⚡
Machine Level Language (MLL) — मशीन स्तरीय भाषा
0 और 1 की भाषा — 1st Generation Language (1GL)
Generation 1 ▼
विस्तृत परिभाषा: Machine Level Language कंप्यूटर की एकमात्र मूल भाषा है जिसे CPU (Central Processing Unit) सीधे समझता और execute करता है। यह Binary Number System (0 और 1) पर आधारित है। 0 का अर्थ है — बिजली नहीं (OFF), 1 का अर्थ है — बिजली है (ON)।

इसे First Generation Language (1GL) कहते हैं। कोई भी translation नहीं होती — CPU सीधे इसे समझता है। लेकिन मनुष्य के लिए यह इतनी कठिन है कि 1950s के बाद इसमें सीधे प्रोग्राम लिखना बंद हो गया।
Machine Language कैसे काम करती है?

Computer का CPU (Processor) Transistors से बना है। प्रत्येक Transistor एक switch है — ON या OFF। ON = 1, OFF = 0। करोड़ों transistors मिलकर जटिल calculations करते हैं।

CPU में Instruction Set Architecture (ISA) होता है — यह define करता है कि कौन सा binary pattern किस operation को execute करेगा। जैसे Intel x86 में 10110000 का मतलब है "Register AL में value रखो।"

→ Hardware का पूर्ण नियंत्रण

Assembly Level Language (ALL) — असेंबली स्तरीय भाषा
Mnemonics-आधारित — 2nd Generation Language (2GL)
Generation 2 ▼
विस्तृत परिभाषा: Assembly Language में 0 और 1 की जगह छोटे-छोटे symbolic codes (Mnemonics) का उपयोग किया जाता है जो याद रखने योग्य होते हैं। जैसे — MOV (Move/Move Data), ADD (Add/जोड़ो), SUB (Subtract/घटाओ), JMP (Jump/कूदो), HLT (Halt/रुको)।

इसे Machine Language में translate करने के लिए एक special program — Assembler — का उपयोग होता है। यह Second Generation Language (2GL) है।
Assembly Language के Components

SP = Stack Pointer

ये registers CPU के कार्य-क्षेत्र हैं।

High Level Language (HLL) — उच्च स्तरीय भाषा
English जैसी सरल भाषा — 3rd, 4th, 5th Generation Languages
Generation 3-5 ▼
विस्तृत परिभाषा: High Level Language ऐसी Programming Language है जो सामान्य अंग्रेजी भाषा जैसी दिखती और पढ़ी जाती है। इसमें मनुष्य-सुलभ शब्द (print, read, if, while, for, class, function) उपयोग होते हैं। CPU इसे सीधे नहीं समझता — इसे Compiler या Interpreter Machine Language में बदलते हैं।

HLL Machine Independent है — एक बार लिखा Python code Windows, Linux, Mac — सब पर चलता है (थोड़े बदलाव के साथ)। यही इसकी सबसे बड़ी शक्ति है।

Language Translators — Interpreter vs Compiler
High Level Language को Machine Language में बदलने वाले program को Language Translator कहते हैं। तीन प्रकार होते हैं: Compiler, Interpreter, और Assembler। यहाँ Compiler और Interpreter की गहन तुलना की गई है।
🔁
Interpreter — इंटरप्रेटर
Line-by-line translation और तत्काल execution
Translator Type 1 ▼
विस्तृत परिभाषा: Interpreter एक ऐसा Language Translator है जो Source Code की एक-एक line को क्रमबद्ध रूप से पढ़ता है, उसे Machine Language में translate करता है, और तुरंत execute करता है। यह पूरे program का एक साथ अनुवाद नहीं करता। हर बार program run करने पर यह दोबारा translate करता है।

नाम का अर्थ: जैसे एक भाषा दुभाषिया (interpreter) बोले गए शब्दों का तुरंत अनुवाद करता है — वैसे ही यह program के instructions का तुरंत अनुवाद करता है।

Compiler — कंपाइलर
पूरे program का एक साथ अनुवाद → Executable file
Translator Type 2 ▼
विस्तृत परिभाषा: Compiler एक ऐसा Language Translator है जो पूरे Source Code को एक साथ पढ़ता है, सभी errors ढूंढता है, और पूरे program को Machine Language में translate करके एक Executable File (.exe, .out) बनाता है। यह Executable File बाद में बिना Compiler के directly run की जा सकती है।

Compilation एक बार होती है। उसके बाद हर बार program directly Machine Code के रूप में चलता है — इसलिए Compiler-based programs बहुत तेज होते हैं।
Compiler की Compilation Process — विस्तार से
1
Lexical Analysis (शाब्दिक विश्लेषण)
Source code को Tokens में तोड़ता है। Keywords (int, if, for), Identifiers (x, y, main), Operators (+, -, =), Literals (5, "hello") आदि identify करता है।
↓
2
Syntax Analysis (वाक्य-विश्लेषण)
Grammar rules check करता है। जैसे if statement के बाद condition होनी चाहिए। Parse Tree बनाता है। यहाँ Syntax Errors मिलती हैं।
↓
3
Semantic Analysis (अर्थ-विश्लेषण)
Type checking — int variable में string assign करने की कोशिश → Error। Undeclared variable use → Error। Logic की जाँच।
↓
4
Intermediate Code Generation
Platform-independent intermediate code बनाता है। Three-Address Code या Bytecode। Optimization के लिए तैयार।
↓
5
Code Optimization
Unnecessary code हटाना, loops को efficient बनाना, constant folding। जैसे x = 2+3 → x = 5 (compile time पर ही)।
↓
6
Code Generation → .exe / .out
Final Machine Code generate होता है। Linker libraries जोड़ता है। Executable file तैयार! User इसे run कर सकता है।

Operating System — कंप्यूटर का मस्तिष्क
Operating System (OS) वह सर्वाधिक महत्वपूर्ण System Software है जो Computer Hardware को manage करता है, Resources allocate करता है, और उपयोगकर्ता को कंप्यूटर से communicate करने का माध्यम देता है।
Operating System की परिभाषा: OS वह System Software है जो — Hardware Resources (CPU, RAM, Disk, I/O) को manage करता है, Application Programs को चलाने का environment देता है, और User तथा Hardware के बीच Interface (सेतु) का काम करता है। इसे Resource Manager भी कहते हैं।

OS के बिना: कंप्यूटर केवल एक बिजली से चलने वाला metal box है। OS ही उसे "intelligent machine" बनाता है।
OS के मुख्य कार्य (Functions of OS)
⚡ Process Management
एक साथ कई programs (processes) चलाना। CPU का time अलग-अलग processes में बाँटना — CPU Scheduling। Deadlock prevent करना। Process create/terminate करना।
💾 Memory Management
RAM को अलग-अलग programs में allocate करना। Virtual Memory — RAM से अधिक memory simulate करना (Swap File)। Memory Leak रोकना। Program बंद होने पर memory free करना।
📁 File System Management
Files और Folders को organize करना। NTFS (Windows), ext4 (Linux), APFS (macOS)। File permissions — कौन पढ़ सकता है, कौन लिख सकता है। Directory structure बनाए रखना।
🔌 Device Management (I/O)
Keyboard, Mouse, Printer, USB — सब I/O Devices को manage करना। Device Drivers के माध्यम से hardware को access देना। Data transfer को coordinate करना।
🔐 Security & Access Control
User Authentication — Login/Password। File Permissions — Owner, Group, Others। Firewall। Virus से सुरक्षा। बिना permission किसी file को access नहीं।
🖥️ User Interface (UI)
GUI (Graphical) — Windows, Icons, Mouse। Desktop, Taskbar, Start Menu। CLI (Command Line) — Terminal, Command Prompt। Commands type करके काम करना।
🌐 Network Management
Internet Connection, Network Interface manage करना। TCP/IP Protocols implement करना। File Sharing, Printer Sharing। WiFi, Ethernet configuration।
⏱️ Job Scheduling
Batch jobs schedule करना। Priority-based processing। Foreground (user देख रहा है) vs Background (पीछे चल रहा है) processes। Windows Task Scheduler, Linux Cron Jobs।

Memory Units — डेटा मापने की इकाइयाँ
Computer में Data को Binary System (0 और 1) में store किया जाता है। इसे मापने के लिए Bit से लेकर Yottabyte तक की विशेष इकाइयाँ होती हैं। ये सभी 2 की शक्तियों (powers) में बढ़ती हैं।
Memory क्यों महत्वपूर्ण है? Computer में सब कुछ — Documents, Photos, Videos, Programs, OS — Binary Data के रूप में Memory में store होता है। Memory जितनी अधिक = अधिक data रख सकते हैं।

दो प्रकार की Memory:
→ Primary Memory (RAM): अस्थायी, तेज, CPU सीधे access करता है। Program चलने पर use होती है।
→ Secondary Memory (HDD/SSD): स्थायी, धीमी, बड़ी। Files permanently store होती हैं।

Computer Memory की सबसे छोटी इकाई — 0 या 1
इकाई 1 ▼
Bit क्या है? Binary Digit — Bit कहलाता है। यह Computer की सबसे छोटी Memory Unit है। एक Bit का मान केवल 0 (शून्य/OFF) या 1 (एक/ON) हो सकता है।

Bit का भौतिक अर्थ: एक Transistor जो ON (1) या OFF (0) हो सकता है। आधुनिक CPU में अरबों Transistors होते हैं — जैसे Apple M4 chip में 28 Billion Transistors!
🔌 भौतिक representation
RAM में: Capacitor — charged (1) या discharged (0)
Hard Disk में: Magnetic field — North (1) या South (0)
SSD में: Flash cell — electrons की उपस्थिति/अनुपस्थिति
Optical Disk में: Land (flat=0) या Pit (hole=1)
📊 Bit के उपयोग
Network Speed: Mbps, Gbps (bits per second)
Color Depth: 8-bit, 16-bit, 32-bit color
CPU Architecture: 32-bit vs 64-bit processor
Encryption: 128-bit, 256-bit AES key
Audio: 16-bit, 24-bit audio quality
Nibble — 4 Bits

Nibble = 4 Bits (आधा Byte)। 4 Bits = 16 possible values (2⁴ = 16, यानी 0000 से 1111)। Hexadecimal का एक digit (0-9, A-F) = 1 Nibble। Memory addresses हेक्साडेसिमल में लिखे जाते हैं।

8 Bits = 1 Byte — Computer Memory की मूल इकाई
इकाई 2 ▼
Byte क्या है? 8 Bits मिलकर 1 Byte बनाते हैं। Byte को Computer Memory की मूल (Basic) इकाई माना जाता है। 1 Byte में एक Character (अक्षर, अंक, symbol) store किया जा सकता है।

1 Byte = 8 Bits = 256 अलग-अलग values (2⁸ = 256, यानी 0 से 255)। इसी 0-255 range में ASCII characters — सभी letters, digits, symbols — encode होते हैं।

Kilobyte (KB) — किलोबाइट
1024 Bytes = 1 KB
इकाई 3 ▼
Kilobyte (KB): 1024 Bytes = 1 Kilobyte।

1024 क्यों — 1000 क्यों नहीं? Computer Binary System (Base 2) में काम करता है। 2¹⁰ = 1024, जो 1000 के सबसे करीब है। इसलिए Computer Science में 1 KB = 1024 Bytes। लेकिन Hard Drive manufacturers 1 KB = 1000 Bytes (Decimal) मानते हैं — इसीलिए 1 TB HDD खरीदने पर OS में कम space दिखता है!

IEC Standard (2000): इस confusion से बचने के लिए: 1 KiB (Kibibyte) = 1024 Bytes, 1 KB (Kilobyte) = 1000 Bytes। लेकिन common usage में 1 KB = 1024 Bytes ही माना जाता है।
